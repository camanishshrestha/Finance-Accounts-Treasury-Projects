## 🎯 Project Objective

When I joined Sanima Group as the **Head of Internal Audit & Compliance**, the group of **9 companies** did not have a **formal, standardized Internal Audit Manual**. Without a manual, internal audit activities lacked **consistency, structured methodology, and documented procedures** — making it difficult to ensure uniform audit quality across all 9 companies. I designed and authored the **Internal Audit Manual 2025** from scratch — creating the definitive reference document governing all internal audit activities across the entire Sanima Group.

> ⚠️ *Note: All company-specific procedures, risk assessments, and proprietary manual content remain strictly confidential. Only the general structure, methodology, and framework are described here.*

### ✅ Step 4: Risk Assessment & Audit Universe

- Designed the **Risk Assessment Framework** for identifying and prioritizing auditable areas:

#### 📋 Audit Universe Mapping

- Mapped the **complete audit universe** — every auditable entity and process across 9 companies:

| Category | Auditable Areas |
|----------|----------------|
| **Financial** | Accounts payable, accounts receivable, payroll, fixed assets, treasury, tax, financial reporting |
| **Operational** | Procurement, inventory, project management, contract management, plant operations |
| **Compliance** | NRB directives (where applicable), Companies Act, Labor Act, Tax Act, environmental regulations |
| **IT** | Access controls, CBS/ERP systems, network security, data backup, change management |
| **Governance** | Board governance, committee functioning, related party transactions, conflict of interest |
| **HR** | Recruitment, performance management, training, leave management, separation |

#### 📋 Risk Scoring Methodology

- Designed a **risk scoring matrix** for each auditable area:

| Risk Factor | Weight | Scoring Scale |
|-------------|--------|--------------|
| **Financial Impact** | 25% | 1 (Low) to 5 (Very High) |
| **Regulatory Risk** | 20% | 1 (Low) to 5 (Very High) |
| **Operational Complexity** | 15% | 1 (Simple) to 5 (Very Complex) |
| **Control Environment** | 20% | 1 (Strong) to 5 (Weak) |
| **Time Since Last Audit** | 10% | 1 (Recent) to 5 (Never Audited) |
| **Management Concern** | 10% | 1 (Low) to 5 (High) |

- Composite Risk Score = Σ (Risk Factor Score × Weight)


- Ranked all auditable areas by composite risk score — **highest risk areas audited first**

### ✅ Step 5: Annual Audit Plan Development Methodology

- Documented the **annual audit plan development process**:

#### 📋 Annual Audit Plan Process

| Step | Activity |
|------|----------|
| **1** | Update **audit universe** — add new entities, processes, or risk areas |
| **2** | Conduct **annual risk assessment** — reassess risk scores for all auditable areas |
| **3** | Determine **available audit resources** — team size, working days, skill sets |
| **4** | Rank auditable areas by **risk score** — highest to lowest |
| **5** | Allocate audit resources to **highest-risk areas** first |
| **6** | Determine **audit frequency** based on risk rating: |
| | — 🔴 High Risk: Annual audit |
| | — 🟡 Medium Risk: Audit every 2 years |
| | — 🟢 Low Risk: Audit every 3 years or rotation basis |
| **7** | Prepare **draft annual audit plan** — audit engagements, timing, resource allocation |
| **8** | Present to **Audit Committee** for review and approval |
| **9** | Obtain **Board approval** of annual audit plan |
| **10** | Communicate approved plan to **management** |

### ✅ Step 7: Fieldwork & Execution Procedures

- Documented standardized **fieldwork procedures**:

#### 📋 Audit Testing Methods

| Method | Description | When Used |
|--------|-------------|-----------|
| **Inquiry** | Interviewing personnel to understand processes and controls | All audits — initial understanding |
| **Observation** | Watching processes being performed to verify control operation | Physical controls, cash handling, security |
| **Inspection** | Examining documents, records, and physical assets | Document review, asset verification |
| **Reperformance** | Independently re-executing a control or computation | Reconciliations, calculations, approvals |
| **Analytical Review** | Analyzing data relationships and trends to identify anomalies | Financial data, KPI analysis, trend analysis |
| **Confirmation** | Obtaining independent verification from third parties | Bank confirmations, vendor confirmations |
| **Computer-Assisted** | Using data analytics tools to test entire populations | Transaction testing, exception analysis |

#### 📋 Evidence Standards

| Criteria | Requirement |
|----------|-------------|
| **Sufficient** | Enough evidence to support audit conclusions |
| **Reliable** | Evidence from credible, independent sources preferred |
| **Relevant** | Evidence directly relates to audit objective |
| **Useful** | Evidence helps achieve audit purpose |

#### 📋 Working Paper Standards

| Requirement | Details |
|-------------|---------|
| **Reference Number** | Every working paper numbered systematically |
| **Prepared By** | Name and date of preparer |
| **Reviewed By** | Name and date of reviewer |
| **Objective** | Clear statement of working paper purpose |
| **Procedure** | Description of audit procedure performed |
| **Results** | Findings, observations, conclusions |
| **Cross-Reference** | Links to related working papers, audit program, and report |

### ✅ Step 8: Audit Reporting Framework

- Designed the **standardized audit reporting framework**:

#### 📋 Audit Report Structure

| Section | Content |
|---------|---------|
| **Report Header** | Audit title, report number, date, distribution list, confidentiality notice |
| **Executive Summary** | Overall audit opinion, key findings summary, overall risk rating |
| **Audit Scope & Methodology** | Period, scope, standards applied, limitations |
| **Detailed Findings** | Individual findings in structured format (see below) |
| **Summary Dashboard** | Visual summary — findings by risk rating, by area |
| **Acknowledgment** | Thanks to auditee management and staff for cooperation |

#### 📋 Individual Finding Format (5 Cs Model)

Finding #: [Sequential Number]
Title: [Brief descriptive title]
- CRITERIA: What should be happening (policy, standard, regulation, best practice)
- CONDITION: What is actually happening (observed situation)
- CAUSE: Why the gap exists (root cause analysis)
- CONSEQUENCE: What is the risk/impact (financial, operational, compliance, reputational)
- RECOMMENDATION: What should be done to address the finding
- RISK RATING: 🔴 Critical | 🟠 High | 🟡 Medium | 🟢 Low
- MANAGEMENT RESPONSE: Auditee's agreed action plan
- RESPONSIBLE PERSON: Who will implement the action
- TARGET DATE: When the action will be completed


#### 📋 Risk Rating Definitions

| Rating | Definition |
|--------|------------|
| 🔴 **Critical** | Immediate action required — significant financial loss, regulatory breach, or fraud risk |
| 🟠 **High** | Prompt action required — material control weakness or compliance gap |
| 🟡 **Medium** | Action required within reasonable timeframe — moderate risk |
| 🟢 **Low** | Improvement opportunity — minor observation, best practice recommendation |

### ✅ Step 9: Follow-Up & Monitoring Framework

- Designed the **finding follow-up and tracking system**:

#### 📋 Follow-Up Process

| Step | Activity | Timeline |
|------|----------|----------|
| **1** | Issue audit report with management-agreed action plans | Within 10 days of fieldwork completion |
| **2** | Enter all findings into **Audit Finding Tracker** | Same day as report issuance |
| **3** | Send **reminder notifications** to responsible persons | 30 days before target date |
| **4** | Conduct **follow-up verification** — verify whether action was implemented | Within 30 days after target date |
| **5** | Update finding status: | |
| | ✅ **Closed** — action fully implemented and verified | |
| | 🔄 **In Progress** — partially implemented, revised target date agreed | |
| | ❌ **Overdue** — target date passed, no action taken — escalate | |
| **6** | **Escalation** for overdue findings: | |
| | — 30 days overdue → Escalate to Department Head | |
| | — 60 days overdue → Escalate to CEO | |
| | — 90 days overdue → Escalate to Audit Committee / Board | |
| **7** | Report **finding status summary** to Audit Committee quarterly | |

### ✅ Step 10: Quality Assurance & Improvement Program (QAIP)

- Designed the **QAIP** as required by IIA Standards:

#### 📋 QAIP Components

| Component | Description | Frequency |
|-----------|-------------|-----------|
| **Ongoing Internal Assessment** | IA Head reviews every audit engagement for quality — methodology compliance, evidence sufficiency, report quality | Every engagement |
| **Periodic Internal Assessment** | Self-assessment of entire IA function against IIA Standards | Annual |
| **External Assessment** | Independent external quality assessment by qualified assessor | Every 5 years |
| **Continuous Improvement** | Action plans to address QAIP findings — training, process improvements, tool enhancements | Ongoing |
