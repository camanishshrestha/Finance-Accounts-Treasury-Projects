## 🎯 Project Objective

When Nepal transitioned from local GAAP to **IFRS/NFRS**, every bank needed a **comprehensive Accounting Manual** — a single reference document that tells every finance team member exactly **how to account for every type of transaction** under the new standards. Without this manual, different people would interpret IFRS differently — leading to inconsistency, errors, and audit issues. I contributed to the preparation of **IFRS/NFRS-based Accounting Manuals and Policies** for two commercial banks — creating the definitive reference guide for their financial reporting.

> ⚠️ *Note: All institution-specific accounting policies, internal procedures, and proprietary manual content remain strictly confidential. Only general structure and methodology are described here.*

## 🛠️ Step-by-Step: What I Did

---

### ✅ Step 1: Gap Identification — Existing Policies vs. IFRS Requirements

- Reviewed the bank's **existing accounting policies** (local GAAP-based)
- Mapped existing policies against **IFRS/NFRS requirements** — identifying:
  - Policies that needed **complete rewrite** (e.g., financial instruments — IFRS 9 replacing IAS 39)
  - Policies that needed **significant amendment** (e.g., revenue recognition, employee benefits)
  - Policies that were **largely compliant** but needed IFRS-specific language updates
  - **New policies required** — areas not covered under previous GAAP (e.g., IFRS 16 Leases, fair value measurement)
- Prepared a **policy gap matrix** documenting every gap and required action

---

### ✅ Step 2: Chart of Accounts (CoA) Design

- Designed the **IFRS-compliant Chart of Accounts** — the foundation of the entire accounting system:

#### 📋 CoA Design Principles

| Principle | Implementation |
|-----------|---------------|
| **IFRS Alignment** | Account structure mapped to IFRS financial statement line items |
| **NRB Alignment** | Accounts structured to also generate NRB regulatory returns |
| **Hierarchical** | Multi-level structure — Group → Sub-Group → Account → Sub-Account |
| **Comprehensive** | Every type of banking transaction has a designated account |
| **Flexible** | Structure allows addition of new accounts without disrupting hierarchy |
| **Coded** | Systematic numbering convention for easy identification |

#### 📋 CoA Structure

| Code Range | Category | Examples |
|------------|----------|----------|
| **1XXXX** | Assets | Cash, NRB balance, loans, investments, fixed assets, other assets |
| **2XXXX** | Liabilities | Deposits, borrowings, provisions, deferred tax, other liabilities |
| **3XXXX** | Equity | Share capital, reserves, retained earnings, OCI reserves |
| **4XXXX** | Income | Interest income, fee income, FX gains, investment income |
| **5XXXX** | Expenses | Interest expense, staff costs, admin expenses, depreciation, provisions |
| **6XXXX** | OCI Items | Actuarial gains/losses, FVOCI fair value changes, revaluation surplus |
| **7XXXX** | Off-Balance Sheet | LC, BG, commitments, contingencies |

- Each account included:
  - Account code and name
  - Account definition and description
  - Normal balance (debit/credit)
  - Financial statement mapping (which line item it feeds into)
  - NRB reporting mapping (which NRB return it feeds into)

### ✅ Step 3: Accounting Policy Drafting — Financial Instruments (IFRS 9)

- Drafted the **most complex and critical policy** — Financial Instruments under IFRS 9:

#### 📋 IFRS 9 Policy Sections

| Section | Content |
|---------|---------|
| **Classification** | Business model assessment methodology; SPPI test criteria; classification into AC, FVOCI, FVTPL |
| **Initial Recognition** | Recognition criteria; transaction costs treatment; day-1 fair value adjustment (staff loans) |
| **Subsequent Measurement** | Amortized cost using EIR; fair value measurement hierarchy; FVOCI recycling |
| **Impairment (ECL)** | 3-stage model; SICR criteria; 12-month vs. lifetime ECL; PD/LGD/EAD methodology; forward-looking adjustment |
| **Derecognition** | Criteria for derecognition of financial assets and liabilities; pass-through arrangements |
| **Hedge Accounting** | Hedging instruments; hedged items; effectiveness testing; documentation requirements |
| **Reclassification** | Conditions for reclassification between categories; accounting treatment on reclassification |

- For each section, included:
  - **Policy statement** — the accounting policy in clear language
  - **Worked example** — numerical example showing how the policy is applied
  - **Journal entries** — exact debit/credit entries for each scenario
  - **Disclosure requirement** — what must be disclosed in financial statements

---

### ✅ Step 4: Accounting Policy Drafting — Employee Benefits (IAS 19)

- Drafted the **Employee Benefits accounting policy** under IAS 19:

#### 📋 IAS 19 Policy Sections

| Section | Content |
|---------|---------|
| **Short-Term Benefits** | Salary, bonus, leave — recognition when service rendered |
| **Post-Employment — Defined Contribution** | PF/SSF — expense recognized as contribution is made |
| **Post-Employment — Defined Benefit** | Gratuity — actuarial valuation methodology, DBO computation, plan assets |
| **Actuarial Gains/Losses** | Recognized in OCI (not P&L) per IAS 19 revised |
| **Other Long-Term Benefits** | Leave encashment — present value of obligation |
| **Termination Benefits** | Recognition criteria — when entity is demonstrably committed |

- Included:
  - Policy statements for each benefit type
  - Actuarial assumption guidance (discount rate, salary escalation, attrition, mortality)
  - Journal entry templates for each benefit recognition scenario
  - Disclosure checklist

### ✅ Step 5: Accounting Policy Drafting — Income Tax (IAS 12)

- Drafted the **Income Tax accounting policy** under IAS 12:

#### 📋 IAS 12 Policy Framework

| Section | Content |
|---------|---------|
| **Current Tax** | Tax computed on taxable profit per Income Tax Act; recognition as expense |
| **Deferred Tax** | Balance sheet liability method; temporary difference identification |
| **DTA Recognition** | Recognized only when probable future taxable profit available |
| **DTL Recognition** | Recognized for all taxable temporary differences (with exceptions) |
| **Tax Rate** | Applied enacted or substantively enacted rate at reporting date |
| **Presentation** | Current tax — current liability; Deferred tax — non-current asset/liability |

#### 🔹 Banking-Specific Temporary Differences Guide
- Included comprehensive **temporary difference identification guide** specific to banking:
  - ECL provisions (DTA)
  - Depreciation differences (DTL)
  - Staff loan day-1 adjustments (DTA)
  - Actuarial provisions (DTA)
  - Fair value changes (DTL)
  - Bonus provision (DTA — deductible when paid)

### ✅ Step 9: NRB Regulatory Reconciliation Guide

- Included a dedicated section on **NRB regulatory reporting reconciliation**:

#### 🔹 IFRS-to-NRB Mapping Framework
- IFRS financial statements and NRB regulatory returns use **different classification and measurement rules**
- Prepared a **reconciliation guide** mapping:
  - IFRS loan classification ↔ NRB loan classification
  - IFRS ECL provisions ↔ NRB regulatory provisions
  - IFRS capital ↔ NRB regulatory capital
  - IFRS financial statement line items ↔ NRB return line items
- Documented the **higher-of provisioning treatment** — IFRS ECL vs. NRB provisions
- This guide enabled the finance team to **simultaneously comply with both IFRS and NRB requirements**
