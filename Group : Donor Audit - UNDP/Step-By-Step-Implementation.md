## 🎯 Project Objective

UNDP donor audits serve a **fundamentally different purpose** from commercial audits — they are not about profitability but about **accountability**. The core question is: **"Were donor funds used for the intended purpose, in compliance with the grant agreement, and with adequate controls to prevent waste, fraud, and misuse?"** As Team Leader/Team Member, I conducted audit of UNDP-funded projects — verifying that funds were utilized as planned, procurement followed UNDP rules, and implementing partners maintained proper records.

## 🛠️ Step-by-Step: What I Did

### ✅ Step 1: Understanding UNDP Audit Framework

- Studied the **UNDP audit framework** and donor project structure before fieldwork:

#### 📋 UNDP Project Structure

| Component | Description |
|-----------|-------------|
| **Donor** | International funding agency providing financial resources (could be bilateral/multilateral) |
| **UNDP Country Office** | UNDP Nepal — manages and oversees project implementation |
| **Implementing Partner (IP)** | Government agency or NGO/INGO executing the project on the ground |
| **Project Document (ProDoc)** | Master document defining project objectives, activities, budget, timeline |
| **Annual Work Plan (AWP)** | Yearly plan specifying planned activities and budget allocation |
| **FACE Form** | Funding Authorization and Certificate of Expenditure — key document for fund request and reporting |

#### 📋 Key UNDP Policies

| Policy | Relevance |
|--------|-----------|
| **UNDP POPP** | Programme and Operations Policies and Procedures — governing all UNDP operations |
| **HACT (Harmonized Approach to Cash Transfers)** | Framework for transferring cash to implementing partners — micro-assessments, spot checks, audits |
| **IPSAS** | International Public Sector Accounting Standards — accounting framework for UN entities |
| **UNDP Procurement Guidelines** | Rules for procurement of goods and services using UNDP funds |

### ✅ Step 2: Audit Planning & Scope Definition

- Developed the **audit plan** specific to the UNDP engagement:

#### 🔹 Document Review
- Reviewed the **Project Document (ProDoc)** — understanding:
  - Project objectives and expected outcomes
  - Total project budget and funding sources
  - Project duration and timeline
  - Key activities and deliverables
  - Implementing partner structure
- Reviewed the **Annual Work Plan (AWP)** — understanding:
  - Planned activities for the audit period
  - Budget allocation by activity and budget line
  - Expected outputs and milestones

#### 🔹 Scope Definition
- Defined **audit scope**:
  - Period covered (specific fiscal year)
  - Implementing partners included
  - Budget lines and expenditure categories to be tested
- Identified **key audit risks**:
  - Expenditure not aligned with approved budget
  - Procurement non-compliance
  - Unsupported or inadequately documented expenditure
  - Asset mismanagement
  - Unauthorized budget revisions
  - Weak internal controls at implementing partner level

---

### ✅ Step 3: Expenditure Verification — Planned vs. Actual

- This was the **core audit procedure** — verifying that actual expenditure was in line with planned/approved budget:


#### 🔹 Variance Analysis
- Analyzed variances:
  - **Under-spending** — funds not utilized, activities not implemented
  - **Over-spending** — expenditure exceeding approved budget (unauthorized)
  - **Budget line transfers** — funds moved between budget lines without proper approval
- Verified that **budget revisions** (if any) were properly authorized by UNDP and documented

---

### ✅ Step 4: Expenditure Vouching — Transaction-Level Testing

- Selected **sample expenditure transactions** and performed detailed vouching:

#### 📋 Expenditure Vouching Checklist

| Check | Verification Performed |
|-------|----------------------|
| **Authorization** | Expenditure approved by authorized project personnel before incurring |
| **Budget Alignment** | Expenditure falls within approved AWP budget line and activity |
| **Supporting Documents** | Adequate supporting documentation available: |
| | — Invoice / bill from vendor |
| | — Purchase order / contract |
| | — Delivery receipt / goods received note |
| | — Service completion certificate |
| | — Attendance sheets (for training/workshops) |
| | — Travel authorization and travel report (for travel) |
| | — Bank payment evidence |
| **Eligibility** | Expenditure is eligible per UNDP project guidelines — not personal, not duplicate, not retroactive |
| **Reasonableness** | Amount is reasonable for the goods/services received |
| **Accounting Accuracy** | Correctly recorded in books — correct amount, account, period |
| **FACE Form** | Expenditure reported in FACE form submitted to UNDP |

#### 🔹 Identified Exceptions
- Identified instances of:
  - **Unsupported expenditure** — transactions without adequate documentation
  - **Ineligible expenditure** — costs not permitted under project agreement
  - **Duplicate payments** — same invoice paid twice
  - **Misclassified expenditure** — recorded under incorrect budget line

---

### ✅ Step 5: Staff Cost Verification

- Verified **project staff costs** — often the largest budget component:

#### 🔹 Verification Activities
- Verified **staff contracts** — position, salary, contract period, terms and conditions
- Verified **salary payments** — monthly salary records, bank transfer evidence
- Verified **time allocation** — staff working on multiple projects must allocate time proportionally
- Checked **salary reasonableness** — compared against UNDP salary scales and local market rates
- Verified **staff benefits** — leave encashment, insurance, allowances — per contract terms
- Identified any **ghost employees** — staff on payroll who don't actually work on the project

---

### ✅ Step 6: Travel Expenditure Verification

- Verified **travel and per diem expenditure**:

#### 📋 Travel Verification Checklist

| Check | Verification Performed |
|-------|----------------------|
| **Travel Authorization** | Prior approval for travel from authorized project manager |
| **Travel Purpose** | Travel purpose aligned with project activities |
| **Travel Mode** | Appropriate mode of travel (economy class, public transport where applicable) |
| **Per Diem Rate** | Per diem rate matches UNDP-approved rates for the travel destination |
| **Per Diem Computation** | Days claimed match actual travel dates — verified against boarding passes, hotel bills |
| **Travel Report** | Travel report submitted after completion — describing activities performed |
| **Supporting Documents** | Boarding passes, hotel invoices, vehicle hire receipts, fuel receipts |
| **Double Claim Check** | No duplicate per diem claim for same travel dates |

---

### ✅ Step 7: Procurement Audit

- Verified **procurement compliance** with UNDP procurement guidelines:

#### 📋 Procurement Verification by Threshold

| Procurement Value | Required Process | Verification |
|------------------|-----------------|--------------|
| **Below NPR X** | Direct purchase / petty cash | Receipts, reasonableness |
| **NPR X to NPR Y** | Minimum 3 quotations | Quotation comparison, selection justification |
| **NPR Y to NPR Z** | Formal bidding process | Bid advertisement, bid documents, evaluation committee, evaluation report |
| **Above NPR Z** | Full competitive bidding | Public advertisement, sealed bids, evaluation committee, Board/UNDP approval |

#### 🔹 Procurement Verification Steps
- For each sampled procurement, verified:
  - Correct **procurement method** applied based on value threshold
  - **Minimum quotations** obtained (typically 3 for mid-value procurements)
  - **Bid evaluation** conducted fairly — documented evaluation criteria and scoring
  - **Lowest evaluated bid** selected (or justified deviation documented)
  - **No conflict of interest** — evaluators had no relationship with selected vendor
  - **Purchase order/contract** issued before goods/services delivered
  - **Goods/services received** as specified — quality and quantity verified
  - **Payment** matches contract terms

#### 🔹 Identified Irregularities
- Identified instances of:
  - **Split procurement** — deliberately splitting purchases to stay below bidding threshold
  - **Single-source procurement** without justification
  - **Missing quotations** — fewer than required number obtained
  - **Vendor favoritism** — same vendor repeatedly selected without competitive process

### ✅ Step 8: Asset Verification & Site Visit

- Conducted **physical verification of assets** purchased with UNDP project funds:

#### 📋 Asset Verification Process

| Step | Activity |
|------|----------|
| **1** | Obtained **asset register** maintained by implementing partner |
| **2** | Selected sample of assets for **physical verification** |
| **3** | Visited project sites to **physically inspect assets**: vehicles, computers, furniture, equipment, construction works |
| **4** | Verified asset **existence** — is the asset actually there? |
| **5** | Verified asset **condition** — is it functional, maintained, being used for project purpose? |
| **6** | Verified asset **tagging** — UNDP asset tag affixed with unique identification number |
| **7** | Verified asset **location** — matches asset register location |
| **8** | Verified asset **custodian** — responsible person identified |
| **9** | Checked for **personal use** — assets being used for non-project purposes |
| **10** | Documented findings with **photographs** |

#### 🔹 Identified Asset Issues
- Identified instances of:
  - Assets **not found** at registered location
  - Assets in **poor condition** — not maintained
  - Assets **used for personal purposes** — not project-related
  - Assets **not tagged** — missing UNDP identification
  - Assets **not in asset register** — purchased but not recorded

---

### ✅ Step 9: FACE Form & Financial Reporting Verification

- Verified the **FACE (Funding Authorization and Certificate of Expenditure) form** — the key financial reporting document between implementing partner and UNDP:

#### 🔹 FACE Form Components
- FACE form reports:
  - Funds received from UNDP
  - Expenditure incurred during the period
  - Balance of funds (received – spent)
  - Request for additional funds (if needed)

#### 🔹 FACE Form Verification
- Verified:
  - FACE form expenditure figures **match accounting records**
  - FACE form expenditure is **supported by vouchers and documentation**
  - FACE form is **signed by authorized signatories**
  - FACE form is **submitted within prescribed timeline**
  - Fund balance reported is **accurate** — reconciled with bank balance
- Reconciled **cumulative FACE form expenditure** with **total project expenditure** in books

---

### ✅ Step 10: Audit Report Preparation

- Prepared the **donor audit report** in compliance with UNDP audit reporting format:

#### 📋 Audit Report Structure

| Section | Content |
|---------|---------|
| **Auditor's Opinion** | Opinion on whether expenditure was incurred in accordance with project agreement and UNDP policies |
| **Basis for Opinion** | Summary of audit scope, methodology, and standards applied |
| **Statement of Expenditure** | Audited statement of expenditure — by budget line, showing approved vs. actual |
| **Findings & Observations** | Specific audit findings: |
| | — Ineligible expenditure (if any) — amount and nature |
| | — Unsupported expenditure — amount and nature |
| | — Procurement non-compliance |
| | — Asset management deficiencies |
| | — Internal control weaknesses |
| **Recommendations** | Specific recommendations for each finding |
| **Management Response** | Implementing partner's response and corrective action plan |

#### 🔹 Financial Impact Reporting
- Reported **financial impact** of findings:
  - **Questioned costs** — expenditure that appears ineligible or unsupported — may need to be refunded to donor
  - **Non-financial observations** — control weaknesses, process improvements
- Submitted audit report to **UNDP Country Office** within prescribed deadline
